# Чекліст для продавця: повернення, обмін, гарантія

**Коли використовується:** при будь-якому зверненні клієнта після покупки.

**Відповідальний:** продавець, КЗ, адміністратор.

---

## 5 питань за 30 секунд

- [ ] **Що хоче клієнт?** Повернення коштів, обмін чи розгляд дефекту.
- [ ] **Яка категорія товару?** Звичайний товар чи білизна / піжами / купальники.
- [ ] **Чи є ознаки дефекту?** Якщо так — це вже не стандартний обмін / повернення.
- [ ] **Чи вкладається звернення в строк?** Для гарантії та вузького винятку по обміну — 14 днів.
- [ ] **Чи товар не був у використанні?** Для обміну це критична умова.

---

## Рішення в одну лінію

::: tabs
== Повернення

- Якщо це стандартний товар належної якості — дійте за загальним порядком магазину.
- Якщо це білизна / піжами / купальники — **кошти не повертаються**.

== Обмін

- Для стандартного товару — перевірте строк, стан товару і підтвердження покупки.
- Для білизни / піжам / купальників — обмін можливий **лише як вузький виняток**:
  - до 14 днів;
  - подарунок / складність підбору розміру;
  - товар не був у використанні;
  - збережено товарний вигляд.

== Гарантія

- Якщо клієнт говорить про прихований дефект — перевірте дату покупки.
- Якщо строк діє і дефект схожий на виробничий — переводьте в гарантійний порядок.
- Якщо причина дефекту спірна — експертиза.
:::

## Готові фрази

**Якщо це білизна / піжами / купальники і клієнт просить повернення:**

> Повернення коштів для цієї категорії товару не здійснюється. У вузькому винятковому випадку ми можемо перевірити лише можливість обміну, якщо товар не був у використанні і звернення відбулося в межах 14 днів.

**Якщо клієнт говорить про дефект:**

> Щоб правильно визначити порядок дій, нам потрібно перевірити дату покупки і характер недоліку. Якщо це гарантійний випадок, ми оформимо звернення належним чином.

**Якщо клієнт тисне:**

> Я озвучую порядок, який діє для цього типу звернення. Якщо є ознаки дефекту або спірна ситуація, ми переведемо звернення у відповідний процес.
