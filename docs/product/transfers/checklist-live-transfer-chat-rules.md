# Чекліст комунікації для LIVE-переміщення

**Коли використовується:** коли магазин подає або обробляє LIVE-запит у чаті.

**Відповідальний:** директор магазину, КЗ, відповідальна особа магазину.

---

## Чекліст

- [ ] LIVE-запит оформлено під реального клієнта.
- [ ] У 1С створено **«Замовлення покупця»**.
- [ ] У повідомленні вказано товар, кількість, магазин-донор, магазин-одержувач і строк резерву.
- [ ] У чаті використовуються тільки факти, підтвердження наявності, підтвердження відправки або коректна причина відмови.
- [ ] Не використовуються формулювання «нам теж потрібен», «під комплект», «плануємо продати».
- [ ] Якщо є спір, питання передано на ескалацію, а не вирішується емоційно в чаті.

---

> [!NOTE]
> Якщо хоча б один пункт не виконано — LIVE-запит треба або дооформити, або передати на рівень керуючого / модератора.

## Коротка версія для директорського чату

> LIVE-переміщення оформлюється тільки під реального клієнта через «Замовлення покупця» в 1С. У чаті працюємо лише фактами: наявність, відправка або коректна причина відмови. Аргументи «нам теж потрібно», «під комплект» чи «плануємо продати» не приймаються. Якщо є спір — питання ескалується, а не обговорюється емоційно.
