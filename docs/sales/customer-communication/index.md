# Правила комунікації з клієнтами

> [!WARNING]
> Цей підрозділ має статус `draft`. Не є офіційним до підтвердження редактором.

## Призначення підрозділу

У цьому підрозділі описуються ручні сценарії комунікації з клієнтами для реактивації, утримання та повернення в магазин. Підхід будується не на масовій розсилці «для всіх однаково», а на пріоритеті цінності клієнта, сили оффера та очікуваного результату.

## З чого починати

Початковим документом для команди є:

- [Регламент: Сегментована комунікація з клієнтами](/sales/customer-communication/reg-segmented-customer-communication)

Цей документ задає:

- базові принципи відбору клієнтів;
- єдині правила для команди;
- матрицю сценаріїв;
- логіку сегментів, офферів і рівнів контакту.

## Ключові принципи розділу

- Не застосовуємо однаковий ручний підхід до всіх клієнтів.
- Пріоритет надаємо цінним сегментам і клієнтам з високим потенціалом повернення.
- Перед контактом перевіряємо актуальне джерело даних для конкретного сценарію.
- Формат контакту обираємо за цінністю клієнта: `дзвінок + повідомлення` / `тільки повідомлення` / `без ручної обробки`.
- Якщо бракує даних або доступу до інструменту — ескалуємо питання керівнику, а не імпровізуємо.
